Structure de la base de données


Les différentes tables et leur structures :


Table régions

    • _id : identifiant de la région : int
    • nom : nom de la région : string


Table départements

    • _id : identifiant du département : double
    • nom : nom du département : string
    • code : code du département : int
    • _id_région : identification de la région à laquelle il appartient : double
    • contours: contient les informations de tracé : array


Table villes        

    • _id : identifiant de la ville : int
    • nom : nom de la ville : string
    • cp : code postal de la ville : string
    • pop : population de la ville : double
    • lat : latitude de la ville : double
    • lon : longitude de la ville : double
    • _id_dept : identification du département à laquelle elle appartient : double

       

Tables contours

    • _id : identifiant du contours : double
    • nom : nom du contours : string
    • poly : dessin du contours : array


Tables users

    • _id : identifiant unique de l'utilisateur : int
    • login : nom de l'utilisateur : string
    • mail : mail de l'utilisateur : string
    • mdp : mot de passe de l'utilisateur : string
    • profil : profil de l'utilisateur : string


Tables readonly :

    • _id : identifiant du champ
    • table : table du champ : string
    • champs : nom du champ immuable : string
